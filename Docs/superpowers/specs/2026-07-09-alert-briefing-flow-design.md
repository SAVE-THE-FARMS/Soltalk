# 알림 브리핑 & 조치 흐름 설계

날짜: 2026-07-09 / 대상: frontend only (백엔드 변경 없음)
레퍼런스: `Docs/2026.07.09_01_[Ref]_대시보드_인터페이스_참고.md` (반달허브 — 중장년 친화 UI)

## 목표

알림함에서 "조치"를 누르면 장비가 소리 없이 바로 조작되는 대신, 사용자가
**상황을 브리핑받고 → 이해하고 → 직접 조치**하는 흐름으로 바꾼다.
(고령 농가 사용자가 "무슨 일이 왜 일어났고 뭘 하면 되는지"를 한 화면에서 인지)

## 변경 사항

### 1. 알림함 (NotificationInbox)
- 버튼 "조치" → **"조치하러 가기"**
- 클릭 시: 장비 조작 없음. 알림함 닫힘 + 해당 온실 상세 화면으로 이동
  (`App`에 `handleGoToGreenhouse(greenhouseId)` 추가: inbox 닫기 + view=dashboard + 온실 선택)

### 2. 온실 상세 (GreenhouseDetail) — 경고/위험 시 브리핑 화면
위에서부터:
1. **상태 카드** — 상태색 배경, 큰 글씨: "⚠️ 습도 82% — 주의가 필요해요" (critical 은 🚨 + 빨강)
2. **📋 현재 상황** — 습도(임계값 대비)·온도 현재값
3. **⚠️ 이대로 두면** — 위험 설명 (습도 초과 → "곰팡이병이 생겨 작물이 상할 수 있어요")
4. **💡 조치 방법** — 권장 조치 설명 ("창문을 열면 바깥 공기가 들어와 습도가 내려가요")
5. **큰 조치 버튼** — 화면 폭 가득, 강조색, 큰 글씨 "🪟 창문 열기" (터치 영역 크게)
6. 조치 직후: 버튼 자리에 "✅ 조치 완료 — 습도가 내려가는 중이에요" (기존 3초 폴링으로 그래프·수치 실시간 갱신)

- 정상 상태 온실: 브리핑 섹션 없이 기존 그래프 화면 유지
- 브리핑 문구 매핑은 `lib/labels.js` 에 상수로 (조치 종류별 위험/설명 텍스트)

### 3. 유지
- 상단 critical 배너의 "바로 조치"는 그대로 (긴급은 원클릭)
- 대시보드 AlertBanner 클릭 → 상세 이동 (기존 동작, 이제 브리핑이 보임)

## 데이터

전부 기존 API 로 충분: `farm.greenhouses[].{status, humidity, temperature, reason,
activeAlert{id, action, message}}` + `/api/state/{id}` 의 `current_values`.
위험/조치 설명 문구는 프론트 상수 (알림 종류가 습도 하나뿐인 데모 규모).

## 검증

Playwright 로 리셋 → 알림함 → "조치하러 가기" → 브리핑 확인 → 조치 버튼 →
완료 문구 + 습도 하강 + 경고 해제까지 e2e 확인.
